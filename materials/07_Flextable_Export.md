# Chapter 7 — Exporting and MS Word Styling with Flextable

While `{gtsummary}` renders perfectly in HTML for interactive environments, clinical trials almost exclusively mandate outputs as RTF or MS Word (`.docx`) documents with strict pagination, margins, headers, and footers.

The primary bridge between `{gtsummary}` and MS Word is the **`{flextable}`** package, combined with the **`{officer}`** package for document-level styling.

## 7.1 Converting to Flextable

To begin styling for Word, you must first convert your `gtsummary` object into a `flextable` object using `as_flex_table()`.

```r
library(flextable)
library(gtsummary)

# 1. Create standard gtsummary table
my_table <- tbl_ard_summary(cards = my_ard, by = TRT01A)

# 2. Convert to flextable
flex_tbl <- my_table %>% as_flex_table()
```

Once it is a `flextable` object, `{gtsummary}` functions no longer apply. You must use `{flextable}` functions.

---

## 7.2 Adding Headers and Footers

In clinical tables, the top-left usually contains the sponsor name and protocol, and the top-right contains the page number.

`{flextable}` allows you to inject literal lines at the top or bottom of the table object itself using `add_header_lines()` and `add_footer_lines()`.

```r
flex_tbl <- flex_tbl %>%
  add_header_lines(values = c("Sponsor: ACME Pharma", "Protocol: 123-ABC", "Table 14.1.1: Demographics")) %>%
  add_footer_lines(values = c("Note: Percentages are based on N.", "Generated by: R-4.4.0"))
```

*Note:* These add rows to the table structure itself. If you want true MS Word Document headers (the ones embedded in the top margin of the Word file that repeat on every page automatically), you use `{officer}` document rendering instead.

---

## 7.3 Advanced Document Layouts (`{officer}`)

By default, `flextable::save_as_docx(flex_tbl, path = "out.docx")` spits out a standard portrait document with default margins. Clinical tables (especially AE tables with many columns) almost always require **Landscape orientation** and custom margins.

To achieve this, we use the `{officer}` package to define a **section property** (`prop_section`).

### 1. Defining Margins and Orientation

```r
library(officer)

# Define a landscape page with 1-inch margins
my_section <- prop_section(
  page_size = page_size(orient = "landscape"),
  page_margins = page_mar(bottom = 1, top = 1, right = 1, left = 1)
)
```

### 2. Saving with Section Properties

Instead of relying on the defaults of `save_as_docx()`, you pass the `pr_section` argument to enforce your layout.

```r
save_as_docx(
  flex_tbl,
  path = "demographics_landscape.docx",
  pr_section = my_section
)
```

## 7.4 Multi-Table Documents

Often, you don't use `save_as_docx()` directly. If you want to generate a massive clinical study report (CSR) containing multiple tables and text, you build an empty `{officer}` Word document in R, inject tables iteratively, and write it out.

```r
# 1. Initialize an empty Word document
doc <- read_docx()

# 2. Add a title paragraph
doc <- body_add_par(doc, "Section 14.1: Demographic Data", style = "heading 1")

# 3. Add the flextable
doc <- body_add_flextable(doc, value = flex_tbl)

# 4. Add a page break
doc <- body_add_break(doc)

# 5. Save the final compiled document
print(doc, target = "clinical_study_report.docx")
```

---

## Chapter 7 Summary

- Use `as_flex_table()` to bridge `{gtsummary}` outputs to the `{flextable}` ecosystem.
- Use `add_header_lines()` and `add_footer_lines()` to append text directly to the top and bottom of the table grid.
- Use `{officer}`'s `prop_section()` alongside `page_size(orient = "landscape")` and `page_mar()` to enforce strict clinical document layouts.
- Pass the section properties to `save_as_docx(pr_section = ...)` or use `read_docx()` to build complex, multi-page reports.
